      About

      Thanks to the recent Javascript renaissance and the spurt of frikin' awesome code it keeps producing, models for writing web apps are changing. As presented in Aaron Quint's jsconf.eu '10 presentation, most of your application can live in the browser as opposed to the server. Couple this with a data store that talks HTTP and you'll begin to experience more expressive and responsive web applications. While far from perfect (don't ask why I implemented this as one horizontal-scrolling page), this page and the code backing it up are meant to start discussion and experimentation for building this type of web app using CouchDB and soca.
